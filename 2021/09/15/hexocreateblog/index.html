hexo+github建立免费的博客网站

说明

搭建个人博客网站听起来是一件很炫酷的事情，觉得很难需要很多的基础，我从基础的小白，花了一两天的时间搭建了属于自己的博客网站。由于是小白，所以使用了github的库来当服务器来搭建自己的个人完全免费的博客网站，现在记录一下搭建的步骤和遇到的问题。个人博客网站https://jercity.github.io/Jercity

准备工作

  • 创建自己的github账号
  • 下载安装node.js，官网可以下载，安装好已集成npm
  • 安装git for windows(或者其他的git客户端)

前面的工作自己可以百度下载注册并了解好。

配置gitbuh的信息

  1. 新建仓库
    在github中，新建自己的仓库，用于存放网页文件的地方。仓库名最好使用‘你的名称.github.io’的形式，不过我注册的时候直接写的名字，后面的域名自己带了.github.io的后缀。

新建之后默认会有主分支，也可以自己新建分支，后续进行部署的时候就是部署在你仓库的分支中(我的就部署在新建的firstTest分支中)。

  1. 域名
    打开新建的仓库，在setting中，找到pages项，生成自己的域名。就是你的仓库名字加.github.io后缀，如果需要就自己注册绑定自己购买的域名，之后访问就是通过这个域名进行的。

  2. 配置SSH key
    这个是很关键的一步，之前我之后使用https的进行部署，发现总是部署失败，无法从本地文件上传到github仓库中，之后换成SSH key部署一次就成功了。

  • 检查SSH key 是否存在
    右键调出Git Bash 窗口,输入
    ls -al ~/.ssh
    如果有id_rsa.pub 或 id_dsa.pub，则已有ssh key
  • 配置SSH key 这里的your github name输入你的GitHub用户名，your github email输入你GitHub的邮箱。这样GitHub才能知道你是不是对应它的账户。
  • 新建SSH key
    打开cmd，在命令行输入
    ssh-keygen -t rsa -C "your_emial@examle.com"
    引号里面是你的邮箱地址，直接按回车即可。
    默认会在相应路径下（/c/Users/Administrator/.ssh/id_rsa）生成id_rsa和id_rsa.pub ，说明密钥生成成功。
  • 生成SSH key
    打开github的主页，进入个人设置 -> SSH and GPG keys -> New SSH key：打开id_rsa.pub文件，里面的信息直接复制生成即可。

hexo安装

hexo是一款很好的支持markdown语法的静态资源网页的部署工具，我之所以选择hexo就其基于node.js，方便部署和安装，相对于github高度整合的Jekyll来说，其用Ruby语法写的部署和安装更为复杂，其hexo的网页的主题个人认为更加好看，不过两款都是非常优秀的网页部署工具。

  1. hexo的安装
    在对应的安装路径下，输入
    $ npm install -g hexo
    即可安装成功，-g表示全局部署，安装成功后才可以使用hexo的命令
  2. 初始化
    新建一个文件夹存放网页文件的，之后这个文件夹就是网页的根目录。我的为hexo文件夹。
    cd hexo
    $ hexo init
    在hexo的文件夹中会生成一些文件，包括node_modules。
  3. 生成静态文件
    输入命令
    hexo g
    其中g是generate的意思，只在生成静态文件，放在public文件夹中，用于部署在github仓库上的文件。
  4. 测试
    输入命令
    hexo s
    s是server的意思，发布本地服务，可以在本地的浏览器访问。
    访问地址默认为http://localhost:4000
    然后打开浏览器，输入http://localhost:4000，会看到成功的网页，属于自己的网页生成了。
    默认的主题是landscape，后期可以更改自己的网页主题，之后再说。

hexo部署

  1. 配置部署环境
    通过之前生成的SSH key来代替github的账号密码来进行上传，首次使用好像要绑定ssh的用户名和邮箱等。

    配置了在如图的地方找到你的clone的ssh。我就是一开始使用HTTPS的文字一直部署失败，现在不能使用了，统一用ssh的方法一下就成功了。
  2. 部署插件安装
    使用npm 命令来安装部署的插件
    npm install hexo-deployer-git --save
    注意，这个命令一定要用git bush打开的窗口赖部署，不然可能会出错，我是用cmd安装插件的，结果有.deploy_git的文件生成，部署失败，网上说删除这个文件就可以，结果我测试不行，最好两种方法都用了，也改了ssh的方法，最好才部署成功。
  3. 部署
    输入命令
    hexo d
    进行部署上传，hexo速度比较快，很快就可以在github中看到上传的网页文件了。注意上传之后会替换你仓库分支的文件，原来的会自动删除，需要及时做好备份。
    在github中查看一下自己的仓库，访问一下自己的域名，查看自己的网站吧。至此，你的个人博客网站就搭建好了，
